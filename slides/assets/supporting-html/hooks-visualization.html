useState

Add state to functional components. Returns current value and a setter function.

const [count, setCount] = useState(0);
‚ö°
useEffect

Handle side effects: data fetching, subscriptions, DOM manipulation.

useEffect(() => {
  // effect
  return () => { /* cleanup */ };
}, [deps]);
üéØ
useRef

Persist values across renders. Access DOM elements directly.

const inputRef = useRef(null);
inputRef.current.focus();
üß†
useMemo

Cache expensive calculations. Only recalculate when dependencies change.

const result = useMemo(() => {
  return expensiveCalc(a, b);
}, [a, b]);
üîó
useCallback

Cache function references. Prevent unnecessary child re-renders.

const handleClick = useCallback(() => {
  console.log(value);
}, [value]);
üîÑ
useReducer

Complex state logic. Like useState but with a reducer pattern.

const [state, dispatch] = useReducer(
  reducer, initialState
);

‚ö†Ô∏è Rules of Hooks

üîù Only Call at Top Level

Don't call Hooks inside loops, conditions, or nested functions.

// ‚ùå Wrong
if (condition) {
  const [x, setX] = useState(0);
}

// ‚úÖ Correct
const [x, setX] = useState(0);
if (condition) { /* use x */ }

‚öõÔ∏è Only Call from React Functions

Call Hooks from React function components or custom Hooks only.

// ‚ùå Wrong
function regularJS() {
  useState(0); // Error!
}

// ‚úÖ Correct
function ReactComponent() {
  const [x, setX] = useState(0);
}

üìù Include All Dependencies

Every value from component scope used in effect must be in deps array.

// ‚ùå Wrong
useEffect(() => {
  console.log(count);
}, []); // Missing 'count'

// ‚úÖ Correct
useEffect(() => {
  console.log(count);
}, [count]);

üîÑ Functional Updates for State

When new state depends on previous, use functional form.

// ‚ùå Might not work
setCount(count + 1);
setCount(count + 1); // Still +1!

// ‚úÖ Always works
setCount(prev => prev + 1);
setCount(prev => prev + 1); // +2!
