The use() Hook - Read Promises in Components!

üìñ Read Promises

Read promise values directly in your component. React suspends until the promise resolves! 

function UserProfile({ userPromise }) {
  const user = use(userPromise);
  return <h1>{user.name}</h1>;
}

üîÄ Conditional Context

Unlike hooks, use() can be called conditionally after early returns! 

function Theme({ skip }) {
  if (skip) return null;

  // Can call after condition!
  const theme = use(ThemeContext);
  return <div>{theme}</div>;
}

React Compiler (Forget)

Automatic memoization - no more useMemo/useCallback everywhere!

‚ùå Before (Manual)

function Component({ items }) {
  const filtered = useMemo(() => 
    items.filter(i => i.active)
  , [items]);
  
  const handleClick = useCallback(() => {
    console.log('clicked');
  }, []);
  
  return <Child 
    items={filtered} 
    onClick={handleClick} 
  />;
}

‚úÖ After (Automatic)

function Component({ items }) {
  // Just write normal code!
  const filtered = items.filter(
    i => i.active
  );
  
  const handleClick = () => {
    console.log('clicked');
  };
  
  // Compiler memoizes automatically!
  return <Child 
    items={filtered} 
    onClick={handleClick} 
  />;
}

üéØ What it does

Analyzes your code at build time and automatically adds memoization where needed.

Cleaner code, fewer bugs from missing dependencies, better performance by default.
